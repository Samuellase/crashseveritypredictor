# Vehicle Crash Severity Predictor

I built this to predict whether a US traffic accident results in low or high severity impact on traffic flow. The dataset has over 2.8 million records — I sampled 500K and turned it into a binary classification problem based on the severity scale used in Delen et al. (2017).

The best model ended up being a stacking ensemble that hit 80% accuracy and a 0.749 AUC on the test set.

## Results

Model | Test Accuracy | AUC
--- | --- | ---
Baseline Random Forest | 78% | —
Tuned Random Forest | 79% | —
XGBoost | 79% | —
Stacking Ensemble | 80% | 0.749

The baseline RF overfit pretty heavily — train accuracy was near 100% while test sat at 78%. Tuning helped marginally. The stacking ensemble (RF + XGBoost + Decision Tree with Logistic Regression as the meta-learner) generalized the best.

## Dataset

[US Accidents (2016–2023)](https://www.kaggle.com/datasets/sobhanmoosavi/us-accidents) from Kaggle. I sampled 500K rows from the full 2.8M, then undersampled the majority class to address the ~63/37 imbalance before training.

The raw data is not included in this repo. Download it from Kaggle and place it at data/raw/US_Accidents_March23.csv.

## What I engineered

Beyond the raw weather and road features, I created several composite features that ended up being the most predictive:

- Temp_Humidity_Index — temperature * humidity / 100, ended up the top feature by importance
- Weather_Severity — composite score combining precipitation, visibility, and wind speed
- Infrastructure_Risk — count of road hazard indicators at the crash location (junction, signal, crossing, stop)
- Rush_Hour — flag for 7-9am and 4-6pm windows
- Dangerous_Hours — flag for 10pm-4am
- Weekend_Night — combined flag for late weekend hours

## Methodology

The pipeline runs in two stages. preprocess.py handles missing value imputation, binary encoding, one-hot encoding of weather conditions (top 10 categories), time feature extraction, and class balancing via undersampling. train.py builds on top of that with the engineered features, runs RandomizedSearchCV on the RF, trains XGBoost separately, then combines everything into the stacking ensemble.

Evaluation covers accuracy, precision, recall, F1, ROC-AUC, and a confusion matrix — all plotted and saved to results/.
